Geocoding

Offline, airgap-ready geocoder.

Cairn turns OpenStreetMap and WhosOnFirst data into a single on-disk bundle, then serves forward search, autocomplete, fuzzy matching, structured queries, and reverse geocoding from one static Rust binary. No cluster. No cloud. No daemon dependencies.

Get started Try it live 
# 1. build a Liechtenstein bundle
$ cairn build \
    --osm liechtenstein-latest.osm.pbf \
    --wof whosonfirst-data-admin-li.db \
    --out bundle

# 2. serve it
$ cairn serve --bundle bundle

# 3. query it
$ curl 'http://localhost:8080/v1/search?q=Vaduz'
{
  "results": [{
    "name": "Vaduz",
    "kind": "city",
    "lon": 9.522,
    "lat": 47.139
  }]
}

Overview

Geocoding turns the name of a place into coordinates and the coordinates of a point into a chain of administrative places. It is what powers the search box on a map, the address autocomplete on a checkout form, and the “you are here” on a satellite image. Pelias, Photon, and Nominatim cover this for online services with Elasticsearch or PostgreSQL behind them.

Cairn is the same thing without the cluster. It targets deployments where the geocoder must run on a single box, in a datacentre with no internet, on a vehicle, or on a developer laptop without warming up Postgres. The runtime is one statically linked binary; the data is one directory of mmap-ready files.

  • Pure Rust at runtime. Single static musl binary.
  • One directory of files; the entire bundle ships in a tar.
  • Tantivy + R*-tree on disk; no cluster, no JVM, no Postgres.
  • Multilingual names, fuzzy matching, layer filters, focus bias.
  • OSM + WhosOnFirst + OpenAddresses + Geonames in one importer.

Install

From a binary release

# Linux x86_64 (musl, statically linked)
$ curl -fsSL -o cairn.tar.gz \
    https://github.com/cairn-geocoder/cairn/releases/latest/download/cairn-v0.0.1-alpha-linux-x86_64-musl.tar.gz
$ tar xzf cairn.tar.gz && cd cairn-*/
$ ./cairn-build --version

From source

$ git clone https://github.com/cairn-geocoder/cairn
$ cd cairn
$ cargo build --release -p cairn-build -p cairn-serve

From a container

$ docker run --rm -p 8080:8080 \
    ghcr.io/cairn-geocoder/cairn:latest

Examples

Forward search

Free-text query. Returns ranked Places with coordinates and admin ancestry. 

$ curl 'localhost:8080/v1/search?q=Vaduz&limit=2'
{
  "query": "Vaduz",
  "results": [
    { "name": "Vaduz", "kind": "city",  "lon": 9.522, "lat": 47.139 },
    { "name": "Vaduz", "kind": "county","lon": 9.520, "lat": 47.144 }
  ]
}

Autocomplete

Prefix match against a per-character ngram index. Cheap enough for keystroke-grain UI. 

$ curl 'localhost:8080/v1/search?q=Vad&mode=autocomplete'

Fuzzy

Edit-distance recovery. Lets users mis-type. Forward mode only; autocomplete stays exact. 

$ curl 'localhost:8080/v1/search?q=vaaduz&fuzzy=2'
# → matches "Vaduz" (1 edit)

Layer filter + focus bias

Restrict to one or more place kinds, and re-rank by distance from a focus point. Useful when the same name appears across kinds (a city and a county both named “Vaduz”). 

$ curl 'localhost:8080/v1/search?q=Vaduz \
    &layer=city \
    &focus.lat=47.165&focus.lon=9.51 \
    &focus.weight=2.0'

Structured search

Field-by-field query. Cairn picks a layer hint based on the finest non-empty field and concatenates the rest into a free-text query. 

$ curl 'localhost:8080/v1/structured \
    ?road=Aeulestrasse&city=Vaduz'

Reverse geocoding

Point-in-polygon against the admin layer; falls back to nearest centroid when the point sits outside any polygon. Results are sorted finest-containing-polygon first. 

$ curl 'localhost:8080/v1/reverse \
    ?lat=47.141&lon=9.523&limit=4'
{
  "source": "pip",
  "results": [
    { "name": "Vaduz",         "kind": "city",    "level": 1 },
    { "name": "Oberland",      "kind": "county",  "level": 1 },
    { "name": "Liechtenstein", "kind": "country", "level": 0 }
  ]
}

Address parsing

A heuristic parser ships in the default build. The libpostal feature swaps in libpostal’s multilingual CRF tagger when its language model is available. 

$ curl 'localhost:8080/v1/parse \
    ?q=Hauptstrasse%2012%2C%2010115%20Berlin%2C%20Deutschland'
{
  "parsed": {
    "postcode": "10115",
    "city":     "Berlin",
    "country":  "Deutschland"
  }
}

API reference

EndpointPurpose
GET /v1/searchForward + autocomplete + fuzzy + layer + focus.
GET /v1/structuredField-by-field address search with auto layer hint.
GET /v1/reversePoint-in-polygon with nearest-centroid fallback.
GET /v1/parseFree-text address → ParsedAddress.
GET /v1/expandFree-text address → normalized variants.
GET /healthzLiveness.
GET /readyzReadiness (returns 503 if no text index).
GET /metricsPrometheus exposition format.

What’s in a bundle

bundle/
├── manifest.toml              schema, source hashes, blake3
├── tiles/<level>/<row>/<col>/<id>.bin   rkyv-archived Place blobs
├── index/text/                tantivy segments (mmap'd at runtime)
└── spatial/
    ├── admin.bin              bincode AdminLayer (polygons + metadata)
    └── points.bin             bincode PointLayer (centroids for fallback)

Bundle integrity is anchored by blake3 hashes in manifest.toml. cairn-build verify recomputes them and refuses any tile or spatial artifact whose hash drifted.

Build sources

SourceFormatFlag
OpenStreetMap*.osm.pbf--osm
WhosOnFirstSQLite per-country bundles--wof
OpenAddressesCSV per-region authoritative addresses--oa
GeonamesTSV populated places--geonames

Mix and match. OSM alone is enough for a working bundle; WhosOnFirst lifts admin polygon quality; OpenAddresses fills addresses where OSM is sparse; Geonames adds populated places where OSM coverage is thin.
